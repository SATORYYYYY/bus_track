# Глава 2. Проектирование системы "Свой Рейс"

## 2.1 Общее описание архитектуры

### 2.1.1 Архитектурный стиль
Система построена по клиент-серверной архитектуре с разделением на три уровня:
- **Уровень представления (Frontend)**: React-приложение
- **Уровень бизнес-логики (Backend)**: Django REST API
- **Уровень данных**: SQLite база данных

### 2.3.2 Физическая модель данных

#### Таблица bookings (Бронирования)

```sql
CREATE TABLE bookings_booking (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER REFERENCES auth_user(id),
    schedule_id INTEGER REFERENCES buses_schedule(id),
    passenger_name VARCHAR(100) NOT NULL,
    passenger_age INTEGER NOT NULL,
    passenger_email VARCHAR(254) NOT NULL,
    passport_number VARCHAR(20) NOT NULL,
    has_luggage BOOLEAN DEFAULT FALSE,
    seat_number INTEGER NOT NULL,
    booking_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'pending',
    booking_code VARCHAR(20) UNIQUE NOT NULL,
    discount_type VARCHAR(20) DEFAULT 'none',
    final_price DECIMAL(10,2)
);
```

#### Таблица buses_schedule (Расписание)

```sql
CREATE TABLE buses_schedule (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    bus_id INTEGER REFERENCES buses_bus(id),
    route_id INTEGER REFERENCES buses_route(id),
    departure_time DATETIME NOT NULL,
    arrival_time DATETIME NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    available_seats INTEGER NOT NULL
);
```

## 2.4 Проектирование пользовательского интерфейса

### 2.4.1 Принципы дизайна для пожилых людей

   - Размер кнопок: минимум 56px высотой

   - Межстрочный интервал: 1.7

## 2.5 API Endpoints

### 2.5.1 Бронирования (Bookings)

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | /api/bookings/create/ | Создание бронирования |
| GET | /api/bookings/{code}/ | Получение бронирования по коду |
| GET | /api/bookings/my/ | Список бронирований пользователя |
| POST | /api/bookings/{id}/pay/ | Оплата бронирования |

### 2.5.2 Рейсы (Buses)

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | /api/buses/schedules/ | Список рейсов |
| GET | /api/buses/schedules/{id}/ | Детали рейса |
| GET | /api/buses/cities/ | Список городов |

## 2.6 Безопасность

### 2.6.1 Аутентификация
- JWT-токены для API
- Сессии для админ-панели
- Хеширование паролей (bcrypt)

### 2.6.2 Авторизация
- Разделение ролей: пользователь, диспетчер, администратор
- Проверка прав доступа на уровне API

## 2.7 Производительность

### 2.7.1 Оптимизации
- Индексы БД на часто запрашиваемые поля
- Кэширование расписания
- Пагинация списков

### 2.7.2 Требования
- Время отклика API: < 500ms
- Загрузка страницы: < 3s
- Доступность: 99.5%
