# 🚌 Свой Рейс - Система бронирования автобусных билетов

Система для продажи билетов на междугородние автобусы с акцентом на удобство использования для пожилых людей и людей с ограниченными возможностями.
## 📋 Описание проекта

"Свой Рейс" - это полноценная веб-система для бронирования билетов на пассажирские автобусные перевозки, разработанная специально для сельской местности. Проект отличается простым и интуитивно понятным интерфейсом с учетом потребностей пожилых пользователей и людей с ограниченными возможностями.

### 🎯 Ключевые особенности

- **Удобный интерфейс для пожилых**: крупные элементы управления, высокий контраст, увеличенные шрифты
- **Система льгот**: автоматический расчет скидок для пенсионеров, студентов, инвалидов и детей
- **Визуальный выбор мест**: интерактивная схема салона автобуса
- **QR-коды билетов**: автоматическая генерация для быстрой проверки
- **Панель диспетчера**: мониторинг рейсов, статистика, управление в реальном времени
- **Адаптивный дизайн**: работа на всех устройствах

## 🚀 Установка и настройка

### Требования

- **Python 3.8+**
- **Node.js 16+**
- **npm 7+**

### 1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd kurs-milena
```

### 2. Настройка бэкенда (Django)

Откройте терминал и перейдите в папку backend:

```bash
cd backend/bus_ticket
```

#### 2.1. Создание виртуального окружения и установка зависимостей

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

pip install -r requirements.txt
```

#### 2.2. Применение миграций базы данных

```bash
python manage.py makemigrations
python manage.py migrate
```

#### 2.3. Загрузка тестовых данных

```bash
python manage.py load_cities        # Загрузка 28 городов России
python manage.py fetch_schedules    # Загрузка расписания (опционально)
```

#### 2.4. Создание суперпользователя

```bash
python manage.py createsuperuser
# Следуйте инструкциям для создания администратора
```

#### 2.5. Запуск сервера

```bash
python manage.py runserver
```

**Доступные URL:**
- API: http://127.0.0.1:8000/api/
- Админ-панель: http://127.0.0.1:8000/admin
- Swagger документация: http://127.0.0.1:8000/api/docs/
- ReDoc документация: http://127.0.0.1:8000/api/redoc/

### 3. Настройка фронтенда (React)

Откройте **новый терминал** и перейдите в папку frontend:

```bash
cd frontend
```

#### 3.1. Установка зависимостей

```bash
npm install
```

#### 3.2. Запуск приложения

```bash
npm start
```

Фронтенд будет доступен по адресу: http://localhost:3000

## 🔌 API Документация

API документировано с использованием OpenAPI 3.0 (Swagger).

### Основные эндпоинты

#### Аутентификация
```
POST   /auth/jwt/create/           # Вход (получение токена)
POST   /auth/users/                # Регистрация
GET    /auth/users/me/             # Профиль пользователя
```

#### Города и рейсы
```
GET    /api/buses/cities/          # Список городов
GET    /api/buses/cities/autocomplete/  # Автодополнение
GET    /api/buses/schedules/      # Поиск рейсов
GET    /api/buses/schedules/{id}/  # Детали рейса
```

#### Бронирования
```
POST   /api/bookings/create/       # Создание бронирования
GET    /api/bookings/{code}/      # Получение по коду
GET    /api/bookings/my/          # Мои бронирования
POST   /api/bookings/{id}/pay/    # Оплата бронирования
```

Полная документация доступна после запуска бэкенда:
- Swagger UI: http://127.0.0.1:8000/api/docs/
- ReDoc: http://127.0.0.1:8000/api/redoc/

## 🗄️ Модели данных

### Booking (Бронирование)
- passenger_name - ФИО пассажира
- passenger_age - Возраст
- passenger_email - Email
- passport_number - Номер паспорта
- has_luggage - Наличие багажа
- seat_number - Номер места
- booking_code - Уникальный код
- discount_type - Тип льготы
- final_price - Итоговая цена
- status - Статус бронирования

### Bus (Автобус)
- number - Номер
- model - Модель
- total_seats - Количество мест
- amenities - Удобства

### City (Город)
- name - Название
- region - Регион
- population - Население
- is_active - Активен

### Route (Маршрут)
- origin - Город отправления
- destination - Город назначения
- distance_km - Расстояние

### Schedule (Расписание)
- bus - Автобус
- route - Маршрут
- departure_time - Время отправления
- arrival_time - Время прибытия
- price - Цена
- available_seats - Свободных мест

### Profile (Профиль)
- full_name - ФИО
- birth_date - Дата рождения
- passport_number - Номер паспорта
- has_luggage - Багаж по умолчанию

## 🐛 Возможные проблемы и решения

### Проблема: Ошибка CORS
**Решение:** Проверьте настройки `CORS_ALLOWED_ORIGINS` в `settings.py`

### Проблема: База данных не создается
**Решение:** Удалите `db.sqlite3` и повторите миграции
```bash
rm backend/bus_ticket/db.sqlite3
cd backend/bus_ticket
python manage.py makemigrations
python manage.py migrate
```

### Проблема: Фронтенд не подключается к бэкенду
**Решение:** Убедитесь, что оба сервера запущены в разных терминалах

### Проблема: react-autosuggest не найден
**Решение:** Установите недостающую библиотеку
```bash
cd frontend
npm install react-autosuggest
```

### Проблема: Неверное время в расписании
**Решение:** Проверьте настройки `TIME_ZONE` в `settings.py`
